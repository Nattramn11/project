# project
 Tehniskā specifikācija
1. Apraksts un Merķis
Merķis: Palīdzēt lietotājam pierakstīt un sekot līdzi dienas darbiem.

Mērķauditorija: Skolēni un studenti.

2. Funkcionālās prasības
F1: Lietotājs var ievadīt un pievienot jaunu uzdevumu.

F2: Lietotājs redz visu uzdevumu sarakstu.

F3: Lietotājs var atzīmēt uzdevumu kā izpildītu.

F4: Lietotājs var izdzēst uzdevumu.

3. Nefunkcionālās prasības
F1: Strādā gan datorā, gan viedtālrunī.

NF2: Vienkāršs un saprotams dizains latviešu valodā.

NF3: Tūlītēja darbība bez lapas pārlādes.

4. Tehnoloģijas
HTML/CSS

JavaScript: Uzdevumu pievienošanas un dzēšanas loģika. Dati glabājas pārlūka atmiņā.

5. Metodoloģija un pamatojums
Metodoloģija: Agile.

Pamatojums:Ļauj ātri izveidot minimālu strādājošu produkta versiju un viegli veikt izmaiņas procesā.

6. Darbu plāns
1nedēļa: Prasību analīze un dizaina skice.

2nedēļa: HTML/CSS vizuālā rāmja izveide.

3nedēļa: JavaScript loģikas programmēšana.

4nedēļa: Testešana, kļūdu labošana un iesniegšana.
